BaseDao的精妙设计

2020年03月17日 1.7k 字 大概 7 分钟
  1. 1. 大致思路
  2. 2. 具体实现

在之前我们就已经学了关于Servlet的抽取，记不清前面的博客是否有提到。原理就是通过建立BaseServlet，业务Servlet去继承BaseServlet，通过反射去调用业务Servlet中的方法。这种方法可以使混乱不堪的Servlet尽可能的化简，从具体功能上到业务上的转变。

而BaseDao也是一种功能抽取的东西，（hibernate中）在我们写关于增删改查的业务时，也许会遇到各种重复的代码，例如保存一个“用户”，与保存一个任意对象到数据库都是调用save()方法，将其转换为持久化对象而已。如果保存不同的对象都要重新写一套方法，无疑是增加了功能的重复性。

大致思路

  • 我们需要创建一个BaseDao接口，将常用的“增删改查”的方法声明写入。
  • 接着再写一个BaseDao的实现类BaseDaoImpl，在实现类中将增删改查功能实现。
  • 接着我们在写Dao层时，例如UserDao，那么我们首先使UserDao继承BaseDao，
  • 再创建UserDao的实现类UserDaoImpl去继承BaseDaoImpl。

通过这一番操作，由于常用的增删改查方法都在BaseDaoImpl中实现了，所以我们在写例如“保存用户”、“删除用户”等方法时，一行代码都不需要写，就可以完成业务。

最重要的是，即使我们是保存“订单”，由于hibernate的特性，我们也可以把订单对象直接传入，BaseDaoImpl依旧可以完成保存等操作。

具体实现

BaseDao设计：

package com.xxx.dao;

import java.io.Serializable;
import java.util.List;

import org.hibernate.criterion.DetachedCriteria;
//通过泛型，可以做到传入任意对象，都能解析的作用
public interface BaseDao<T> {
    //增
    void save(T t);
    //删，传入对象删除
    void delete(T t);
    //删，通过id删除，所有可作为id的对象都实现了Serializable
    void delete(Serializable id);
    //改
    void update(T t);
    //查 根据id查询
    T getById(Serializable id);
    //分页查询
    //查寻符合条件的总记录数
    Integer    getTotalCount(DetachedCriteria dc);
    //查 查询分页列表数据
    List<T> getPageList(DetachedCriteria dc,Integer start,Integer pageSize);
}

BaseDaoImpl设计：

package com.xxx.dao.impl;

import java.io.Serializable;
import java.lang.reflect.ParameterizedType;
import java.lang.reflect.Type;
import java.util.List;

import org.hibernate.criterion.DetachedCriteria;
import org.hibernate.criterion.Projections;
import org.springframework.orm.hibernate5.support.HibernateDaoSupport;

import cn.xxx.dao.BaseDao;

//由于在此，依旧没有确定泛型类型，因此实现类也使用泛型。
//由于是Spring项目，所以继承hibernate模板。
public class BaseDaoImpl<T> extends HibernateDaoSupport implements BaseDao<T> {
    //由于我们在只有在运行时，它BaseDaoImpl才能知道具体的泛型类型，但是在实现方法时我们就要用到。所以我们需要设置在运行时期接受类型的代码。
    private Class clazz;//用于接收运行期泛型类型


    public BaseDaoImpl() {
        //在程序运行时期，例如UserDaoImpl执行，那么它继承了BaseDaoImpl。我们会在泛型中填入User。执行时是UserDaoImpl，那么获取到的父类就是BaseDaoImpl。
        //获得当前类型的带有泛型类型的父类
        ParameterizedType ptClass = (ParameterizedType) this.getClass().getGenericSuperclass();
        //ParameterizedType可获取泛型类型，原返回值是Type顶级接口。
        //获得运行期的泛型类型
        //泛型可能有多个，由于BaseDaoImpl只有一个泛型，所以我们只需要接收第一个泛型。
        clazz = (Class) ptClass.getActualTypeArguments()[0];
    }

    @Override
    public void save(T t) {
        getHibernateTemplate().save(t);
    }

    @Override
    public void delete(T t) {

        getHibernateTemplate().delete(t);

    }

    @Override
    public void delete(Serializable id) {
        T t = this.getById(id);//先取出对象,再执行删除
        getHibernateTemplate().delete(t);
    }

    @Override
    public void update(T t) {
        getHibernateTemplate().update(t);
    }

    @Override
    public T getById(Serializable id) {
        return (T) getHibernateTemplate().get(clazz, id);
    }

    @Override
    public Integer getTotalCount(DetachedCriteria dc) {
        //根据业务来讲，我们会在前台收到查询条件，然后我们将其封装为离线查询对象。在此进行获取其总数据量，那么我们就需要再次设置聚合函数。

        //设置查询的聚合函数,总记录数
        dc.setProjection(Projections.rowCount());
        List<Long> list = (List<Long>) getHibernateTemplate().findByCriteria(dc);
        //但是要注意的是，我们在查询后必须清空聚合函数，因为离线查询对象主要还是作为离线查询使用，主要获取的还是数据集合，如果我们不清空聚合函数，那么在后续查询list集合时，最终会返回“数据总条数”。
        //清空之前设置的聚合函数
        dc.setProjection(null);
        if(list!=null && list.size()>0){
            Long count = list.get(0);
            return count.intValue();
        }else{
            return null;
        }
    }

    @Override
    public List<T> getPageList(DetachedCriteria dc, Integer start, Integer pageSize) {
        //后续...如果前面方法不清空聚合函数，此处返回的list将仅仅是一个集合，其中只有一个数字而已。
        List<T> list = (List<T>) getHibernateTemplate().findByCriteria(dc, start, pageSize);
        return list;
    }

}

关于分页的一些解释：

在上述方法中，我们特意设置了分页查询的相关方法。那么究竟分页功能是如何使用这些方法的呢？

PageBean设计：

package com.xxx.domain;

import java.util.List;

public class PageBean {
    //当前页数,用于计算索引
    private Integer currentPage;
    //总记录数,前端显示，计算总页数
    private Integer totalCount;
    //每页显示条数，用于计算索引
    private Integer pageSize;
    //总页数，前端使用
    private Integer totalPage;
    //分页列表数据
    private List list;
    //设置构造方法，计算总页数以及错误数据修正。
    public PageBean(Integer currentPage, Integer totalCount, Integer pageSize) {
        this.totalCount = totalCount;

        this.pageSize =  pageSize;

        this.currentPage = currentPage;

        if(this.currentPage == null){
            //如页面没有指定显示那一页.显示第一页.
            this.currentPage = 1;
        }

        if(this.pageSize == null){
            //如果每页显示条数没有指定,默认每页显示3条
            this.pageSize = 3;
        }

        //计算总页数
        this.totalPage = (this.totalCount+this.pageSize-1)/this.pageSize;

        //判断当前页数是否超出范围
        //不能小于1
        if(this.currentPage < 1){
            this.currentPage = 1;
        }
        //不能大于总页数
        if(this.currentPage > this.totalPage){
            this.currentPage = this.totalPage;
        }

    }
    //计算起始索引
    public int getStart(){
        return (this.currentPage-1)*this.pageSize;
    }

    public Integer getCurrentPage() {
        return currentPage;
    }
    public void setCurrentPage(Integer currentPage) {
        this.currentPage = currentPage;
    }
    public Integer getTotalCount() {
        return totalCount;
    }
    public void setTotalCount(Integer totalCount) {
        this.totalCount = totalCount;
    }
    public Integer getPageSize() {
        return pageSize;
    }
    public void setPageSize(Integer pageSize) {
        this.pageSize = pageSize;
    }
    public Integer getTotalPage() {
        return totalPage;
    }
    public void setTotalPage(Integer totalPage) {
        this.totalPage = totalPage;
    }
    public List getList() {
        return list;
    }
    public void setList(List list) {
        this.list = list;
    }
}

其中，“当前页数”和“当前页显示多少条数”由前端传入。总页数由构造方法通过“总数据量”与“当前页显示多少条”计算得出。

剩下的“总数据量“和”数据集合“则是BaseDaoImpl的最后两个方法得出。

此时我们将PageBean完全封装。

此后，我们无论使用任意对象的分页，或是任意对象的增删改查，都完全可以不写任何代码去实现。

而如果在业务上需要其他的查询，我们也可以（比如UserDao），在UserDao中进行声明，在UserDaoImpl中进行实现。
