这两天开始简单的整合之前学过的知识，做一个小小的项目。今天的项目中，其中最复杂的应该就是历史记录了。

历史记录

要想做商品的历史记录，首先，我们先思考业务。

我们的业务就是在每点击一个商品，进入商品详情页时，都将该商品的信息记录下来保存在域中。

当我们访问历史记录的页面时，我们就从域中读取。

这个是比较正常的思维。

历史记录，我们可以保存在客户端，也可以保存在服务端。这是两种不同的方法，一般的电商模式可能会将历史记录保存在服务端，因为用户的历史记录如果保存在客户端，那么当用户换设备时无法将数据同步回来。

服务端一般我们可以存在session域当中，或者我们可以将它存在redis中，一般不建议将其持久化到数据库，因为存取速度太慢。

客户端我们也只能选择cookie了。

根据学习的需求，我们将它存在cookie中，我们在每次点击任意商品时，都会进入一个为“商品详情页”提供信息的servlet。那么，我们记录历史肯定也是在这个servlet中进行记录了。

我们不能确定是否是第一次访问商品，因此我们不能确定cookie中是否有数据，因此在每次访问的时候，在记录历史之前，我们先读取cookie。

如果cookie中已经有历史记录，那么我们将现在访问的商品的信息与cookie中读取到的历史信息拼在一起。

最后将其返回给浏览器，这个cookie会在访问任何资源时都携带，因此只需要在为”历史记录页面”提供信息的servlet中读取数据，并转发给页面即可。

记录历史信息详细代码：

package com.xxxx.web.servlet;

import com.xxxx.domain.Product;
import com.xxxx.service.ProductService;

import javax.servlet.ServletException;
import javax.servlet.http.Cookie;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.util.Arrays;
import java.util.LinkedList;
import java.util.List;

public class ProductInfoServlet extends HttpServlet {
    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }

    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        String pid = request.getParameter("pid");
        ProductService service = new ProductService();
        Product product = service.findProductByPid(pid);

        String pids = pid;
        Cookie[] cookies = request.getCookies();
        if (cookies != null) {
            for (Cookie cookie : cookies) {
                if ("pids".equals(cookie.getName())) {
                    pids = cookie.getValue();
                    //将字符串转为LikedList 方便操作
                    String[] pidsQuery = pids.split("-");
                    List<String> tmpList = Arrays.asList(pidsQuery);
                    LinkedList pidsList = new LinkedList(tmpList);
                    //如果集合中已经有该元素则将其取出，将pid添加到集合首部
                    //如果没有，则直接将pid添加到集合首部
                    if (pidsList.contains(pid)) pidsList.remove(pid);
                    pidsList.addFirst(pid);
                    StringBuilder sb = new StringBuilder();
                    for (int i = 0; i < pidsList.size() && i < 7; i++) {
                        sb.append(pidsList.get(i));
                        sb.append("-");
                    }
                    //subString方法：索引值从零开始。根据填入的参数作为起始结束索引的话，得到的新串包含起始索引，不包含结束索引。
                    //从上一个for循环走出后，得到的串会多携带一个分隔符，因此在此去掉。
                    pids = sb.substring(0, sb.length() - 1);
                }
            }
        }
        Cookie cookie = new Cookie("pids", pids);
        cookie.setPath(request.getContextPath());
        cookie.setMaxAge(60 * 60 * 24 * 14);
        response.addCookie(cookie);

        request.setAttribute("product", product);
        request.getRequestDispatcher("/product_info.jsp").forward(request, response);
    }
}

读取历史信息返回给页面详细代码：

package com.xxxx.web.servlet;

import com.xxxx.domain.Product;
import com.xxxx.service.ProductService;
import com.xxxx.vo.PageBean;

import javax.servlet.ServletException;
import javax.servlet.http.Cookie;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.util.ArrayList;
import java.util.List;

public class FindProductListByCidServlet extends HttpServlet {
    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }

    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        String cid = request.getParameter("cid");
        String pageStr = request.getParameter("page");
        if (pageStr == null) pageStr = "1";
        int currentPage = Integer.parseInt(pageStr);
        int currentCount = 12;
        ProductService service = new ProductService();
        PageBean pageBean = service.findProductListByCid(cid, currentPage, currentCount);
        request.setAttribute("pageBean", pageBean);
        request.setAttribute("cid", cid);

        //获取历史记录信息
        List historyProductList = new ArrayList<Product>();
        Cookie[] cookies = request.getCookies();
        if(cookies != null){
            for(Cookie cookie : cookies){
                if("pids".equals(cookie.getName())){
                    String pids = cookie.getValue();
                    String[] strings = pids.split("-");
                    for(int i = 0;i < strings.length;i++){
                        Product product = service.findProductByPid(strings[i]);
                        historyProductList.add(product);
                    }
                }
            }
        }
        request.setAttribute("historyProductList",historyProductList);
        request.getRequestDispatcher("/product_list.jsp").forward(request, response);
    }
}

浏览器返回时的历史记录展示

通过js重载页面

在我们的页面中，“历史记录页面”与“商品展示页面”是在同一个页面上的，也就是说，我们在“商品展示页面”点击任意商品，它会进入到“商品详情页”中，而普通人的思维逻辑是，当需要返回时，按浏览器的返回按钮。

这里就有问题了，我们在进入商品详情页后，如果直接返回到商品展示页面，浏览器会读取本地缓存。不会重新加载页面，这就使我们的历史记录不会即时刷新到页面上，如果想使它即时显示，就必须在返回时重新加载页面。

我们可以在商品详情页上添加按钮，点击按钮返回上一页。但是一般用户可不一定会根据我们的设想操作，他们也许依旧会通过浏览器返回。

我经过查资料，找到一种通过js获取页面是否被浏览器返回的方法。

页面的js代码：

if(localStorage.getItem('state') == 1) {
        localStorage.setItem('state', 0);
        location.href = location.href;
    }

    window.onbeforeunload = function () {
        localStorage.setItem('state', 1);
    };

这段代码的意思是，首先给页面设置一个事件，当页面即将离开时（可能是去往商品详情页，也可能去往其他任意页面），我们都在本地存储一个“状态”。

而当页面加载时，我们都去本地读取这个“状态”，如果能读取到，那么它必定之前被载入过，那么为了防止页面重复加载，我们首先将这个状态取消，然后再使页面重定向到它本身，这就做到了即便是浏览器后退，我们也可以重新加载页面达到正常显示数据的目的。
